# Connect Google Cloud to Ops

<walkthrough-disable-features toc></walkthrough-disable-features>

This connects your Google Cloud project to Ops using **Workload Identity
Federation** — Ops gets short-lived, on-demand credentials, with **no
service account keys** to create, store, or leak. Setup takes about a minute.

## Before you start

Make sure the active Cloud Shell project is the one you want to connect:

<walkthrough-project-setup></walkthrough-project-setup>

You can review exactly what will be created by opening
<walkthrough-editor-open-file filePath="main.tf">main.tf</walkthrough-editor-open-file>.

## Connect your project

In the Ops dashboard, on your connection's page, copy the **setup command**
and paste it here, then run it.

The command sets a few values for your connection and runs
<walkthrough-editor-open-file filePath="setup.sh">setup.sh</walkthrough-editor-open-file>,
which provisions the federation resources with Terraform and reports back to
Ops. It is safe to re-run if anything is interrupted.

It looks like this:

```bash
export OPS_API_URL="..." \
       OPS_CONNECTION_ID="..." \
       OPS_VERIFICATION_TOKEN="..." && \
./setup.sh
```

## Finish

When the command prints **Done**, return to the Ops dashboard. The
connection will show as connected shortly.

<walkthrough-conclusion-trophy></walkthrough-conclusion-trophy>
